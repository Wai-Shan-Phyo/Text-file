Two Types of Web Pages
အဓိကအားဖြင့် Web Pages အမျိုးအစားနှစ်မျိုး ရှိပါတယ်။
(1)Static Web Page
(2)Dynamic Web Page တို့ပဲဖြစ်ပါတယ်။

What is a Static Web Page?
             Static Web Page ဆိုသည်မှာ ဝင်လိုက်သည်နှင့် ဘယ်သောအခါမှ content တွေ မချိန်းပါ ဘူး။ default stable ဖြစ်နေသည့် page ကို ဆိုလိုခြင်း ဖြစ်ပါတယ်။ Static Web Page များ၏ code တွေသည် “pre written code” တွေပဲဖြစ်ပါသည်။ Pre written code တွေဆိုသည်မှာ Programmer တွေ က ကြိုတင်ရေးသားထားသော code တွေဖြစ်ပါသည်။ user တွေအနေနှင့် web page တစ်ခုကို ၀င် လိုက်ချိန်တွင် Web Page ၏ owner ဘက်မှ Programmer တွေကို  content မပြောင်းခိုင်းသရွေ့ Programmer တွေ ရေးသားထားသော Web Page design အတိုင်းပဲ အမြဲ မပြောင်းလဲ ဘဲမြင်နေရမှာကို ဆိုလိုခြင်းဖြစ်ပါတယ်။ ဥပမာ - User တစ်ယောက်သည် Web Page တစ်ခုထဲ၀င် ပြီး button တစ်ခုကိုနှိပ်လိုက်ပါက ထို button ၏ color သည် ပြောင်းလဲသွားမည်ဆိုလျှင် နောက်ထပ် user တစ်ယောက်သည်လည်း ဤ Web Page ထဲ သို့ထပ် ၀င်ပြီး ထို button ကိုနှိပ်မည်ဆိုပါက ပထမ user ကဲ့သို့color ပြောင်းလဲခြင်းကို ထပ်မံတွေ့ရှိရမှာပဲဖြစ်ပါတယ်။ မြင်သာသော static web page ကို ဉပမာပြရမည်ဆိုလျှင် Wikipedia ဖြစ်ပါသည်။ Wikipedia web page အတွင်းရှိ စာသားများသည် ထို web page ၏ owner သို့မဟုတ် ထို web page ကိုရေးသားသူ programmer ဘက်မှ modify မလုပ်မချင်း မပြောင်းလဲဘဲ stable ဖြစ်နေမှာပဲဖြစ်ပါတယ်။

What is a Dynamic Web Page?
      Dynamic Web Page ဆိုသည်မှာ content တွေကတစ်မျိုးထဲဖြစ်မနေဘဲ user(client) က ပေးလာသော request ပေါ်မူတည်ပြီး server ကပြန်ပေးလာသည့် Web Page ကိုဆိုလိုခြင်း ဖြစ်ပါတယ်။ Dynamic Web Page တွေ ၏ code ကို Server Side programming language ဖြင့် ရေးသားထားခြင်းပဲဖြစ်ပါတယ်။ Dynamic Web Page ၏ သဘောတရားကို ပိုမိုမြင်သာအောင် စာရေးသူက လူတိုင်းနှင့် ရင်းနှီးမှုရှိသော ဉပမာ တစ်ခုနှင့် ယှဉ်ပြောပြပါမည်။ ဉပမာ - Facebook ကိုတော့ လူတိုင်းရင်းနှီးပြီးသားဖြစ်ပါလိမ့်မည်။ မိမိ Facebook account ထဲရှိ news feed သည်မိမိ like လုပ်ထားသော Page များမှသတင်း မိမိaccount ထဲတွင်ရှိသော မိမိ Friend များ ၏ သတင်းအများစုသာ ပေါ်နေမည်ဖြစ်ပါသည်။ တခြားသော user များ၏ account ၏ news feed တွင်လည်း အမြဲတစ်သမတ်တည်း ပေါ်နေမည်မဟုတ်ဘဲ အချိန်နှင့်တစ်ပြေးညီ ပြောင်းလဲနေမှာပဲ ဖြစ်ပါတယ်။ Facebook သည် user တွေ၏ search history ကိုအမြဲ record လုပ်ထားလေ့ရှိပြီး database ထဲတွင် user တစ်ယောက်၏ facebook account, username နှင့် password နှင့် သီးသန့်ခွဲပြီး record မှတ်ထားလေ့ရှိပါတယ်။ ထိုနောက် facebook ဘက်ခြမ်းမှ user တစ်ယောက် ၏search history,interest,page like တွေအပေါ်မူတည်၍ အနီးစပ်ဆုံးကြိုက် နှစ်သက်နိုင်ခြေရှိ နိုင်သော information တွေကိုအများဆုံးပြန်ပြပေးလေ့ရှိပါသည်။ အကယ်၍ မိမိသည် Motorcycle Crazy တစ်ယောက်ဖြစ်ပြီး မိမိ၏ accountတွင် search အဖြစ်ဆုံးသည်လည်း Motorcycle ဖြစ်ပါ က Facebook team ဘက်မှ filter ပြန်စစ်ပြီး မိမိ၏ news feed ကိုအများဆုံး ပြန်လည်ပို့ဆောင် ပေးမည့် သတင်းသည် လည်း Motorcycle နှင့်အများဆုံးဆက်နွယ်နေသော သတင်းများသာဖြစ် ပေလိမ့်မည်။ Twitter သည်လည်း information ကို အချိန်နှင့်တစ်ပြေးညီ ပြပေးနေလေ့ရှိသော Dynamic Web Page ဖြစ်ပါသည်။ စာရေးသူတို့ ရှာလိုသမျှတစ်စုတစ်စည်းထဲ အမြဲရှာလို့ရနိုင် သော Google သည်လည်း Dynamic Web Page ဖြစ်သလို Gmail နှင့် Pinterest သည်လည်း Dynamic Web Page များဖြစ်ကြပါတယ်။
             အကယ်၍ Web page တစ်ခုသည် Dynamic Web Page လား Static Web Page လား မသဲ ကွဲဖြစ်နေပါလျှင် builtwith.com  သို့မဟုတ် wappalyzer.com တို့တွင် သွားရောက်၍ မိမိ သိလိုသော Web Page ၏ name သို့မဟုတ် url ကိုထည့်၍ check လုပ်ကြည့်နိုင်ပါသည်။

များသောအားဖြင့် Static Web Page အများစုကို HTML/CSS/JavaScript တွေဖြင့်ရေးသားလေ့ရှိပြီး Dynamic Web Page အများစုကိုတော့ Server Side Programming language တွေဖြစ် သော Java/Servlet/JSP/PHP/Node.js တွေနှင့်အများဆုံးရေးသားလေ့ရှိပါသည်။

Application
Application မှာဆိုရင်လည်း ၂မျိုးရှိပါသည်။
     (1)Standalone application
     (2)Web application
Standalone application ဆိုသည်မှာ Microsoft Word, excel နှင့် antivirus software တွေဖြစ််ပါသည်။ စက်တစ်ခုပေါ်မှာ Install လုပ်ပြီးလျှင် တခြားသောစက်မှ ချိတ်ဆက်သုံးဖို့ အဆင်မ ပြေနိုင်ပေ။ ထိုအရာကို Desktop application, Standalone Application လို့ခေါ်ပါသည်။

Web-Application တိုင်းမှာ front-end နှင့် back-endဆိုပြီးနှစ်မျိုးရှိပါသည်။ Front-end ကို client-side ဟုလဲခေါ်ပါတယ်။ Client-side ၏ အဓိကရည်ရွယ်ချက်သည် data တွေကို browser မှာမြင်နိုင်အောင် Userကိုပြပေးခြင်းပဲဖြစ်ပါတယ်။ Front-end အပိုင်းမှာတော့ HTML, CSS & JavaScript ကိုအသုံးပြုပါတယ်။
Back-end ပိုင်းကို Server Side လို့လဲခေါ်ကြပါတယ်။ Back-end ပိုင်းသည် user မမြင်ရသောအပိုင်းဖြစ်ပါသည်။ User ကပေးလာသော request ကိုယူပြီး server ထဲမှာ user ပေးခဲ့သော request နှင့် တိုက်စစ်ပြီးရလာသော data တွေကို response ပြန်ပေးပါတယ်။ Back-end ပိုင်းမှာဆိုရင် PHP, Java, Python, JavaScript နှင့် အခြား back-end languages များကိုအများဆုံး အသုံးပြုလေ့ရှိကြပါတယ်။ ထို့ကြောင့် Web application  အလုပ်လုပ်ပုံသည် user တစ်ယောက် သည် browser မှ url သုံးပြီးဝင်လာပြီး server ကို requestပို့ ပါသည်။ Server သည်လည်း dataရှာ ပြီး HTML, CSS, JavaScript file နှင့် content page များကို browser ဆီကို response ပြန်ပေးပါသည်။ User မှ ထပ်မံ request ပေးလာပါက server သည် နောက်ထပ် data များကို response ပြန်ပေးသော လုပ်ငန်းစဉ်ကိုသာ ထပ်ခါတလဲလဲလုပ်ပါလိမ့် မည်။
       Web Application ဆိုသည်မှာ install လုပ်စရာမလိုပေ။ ထို Application ကို Web Server ၏ နေရာတစ်နေရာမှာ ထားထားပါသည်။ ထို web-server တွင် Application ကိုထည့် ပေးထား ပါ တယ်။ ထို application ကို internet-access ရှိမယ်ဆိုရင်ကြိုက်နှစ်သက်ရာ device ကနေ browser ရှိရုံဖြင့် လှမ်းသုံးနိုင်မှာပဲဖြစ်ပါသည်။ Web application တစ်ခုကိုသုံးဖို့ရာ Client ဟု ခေါ်သော browser နှင့် Web serverလိုအပ်ပါသည်။ Browser ဆို သည်မှာ အမျိုးမျို: ဖြစ်နိုင်ပါသည်။  ဉပမာ - Chrome, Firefox တို့ပဲဖြစ်ပါသည်။
  Browser သည် UI(User Interface) ပိုင်းအတွက်လိုအပ်ပါသည်။ Web Application ကို ထည့်ထားလို့သော Web Server လည်း လိုအပ်ပါတယ်။ Web Server အနေနှင့်စာရေးသူတို့က Apache Tomcat တို့လို Server ကိုသုံးမှာပဲဖြစ်ပါသည်။ Web Server ပေါ်တင်ထားပြီဆိုလျှင် မိမိ ကြိုက်နှစ်သက်ရာ နေရာကနေ internet-access ရရုံဖြင့်လှမ်းသုံးနိုင်မှာပဲဖြစ်ပါတယ်။
   Client နှင့် Browser ကလည်း တစ်ခုနှင့်တစ်ခုချိတ်ဆက်နေရပါမည်။ ဉပမာ - Computer တစ်လုံးက Data များကိုနောက်တစ်လုံးဆီ ကူးချင်လျှင် Stick သို့မဟုတ် Hard-Disk နှင့် File တွေကို Transfer ကူးလို့ရပေမဲ့ ဒီနေရာမှာ ၎င်းတို့ကိုချိတ်ပေးမှာသည် HTTP(Hypertext Transfer Protocol) ပဲဖြစ်ပါတယ်။

About Server
Web server ဆိုသည်မှာ website တွေ ကို run နိုင်အောင် server software run ထားသော computer ဖြစ်ပါသည်။ အခြေခံအားဖြင့် Web server ဆိုတာကို အလွယ်ဆုံးပြောရရင် internet ပေါ်မှာ website တွေကိုကြည့်လို့ရအောင် လုပ်ပေးသော computer ဖြစ်သည်။ ကျွန်တော်တို့ အသုံးပြုနေသော computer တွေကို server software ကို run ထားရင် server computer ဟုခေါ်လို့ရပါသည်။ Web server သည် HTTP(Hypertext Transfer Protocol) ကိုအသုံးပြုပြီး user သို့ web page များကို ဖော်ပြပေးသည်။ Web Server ၏အဓိကအလုပ်သည် Client ၏ request ကို လက်ခံသည်။ ထို့နောက် control လုပ်တယ်။ နောက်ဆုံးတွင် response ပြန်ပေးသည်။ Control လုပ်သော အခါတွင်လဲ လိုအပ်လျှင် Database နှင့်ပါချိတ်ဆက်ပြီး control လုပ်ပေးသည်။ ကျွန်တော်တို့ facebook,twitter,skype တို့ကိုအသုံးပြုသောအခါ web browser သို့မဟုတ် client  app ကိုသုံးပြီး Login စာမျက်နှာ၌ user ၏ email နှင့် password ကိုရိုက်ထည့်ပေးရသည်။ Client app မှ ရိုက်ထည့်လိုက်သော email နှင့် password ကိုယူပြီး server သို့လှမ်းပြီး request လုပ်သည်။ ထိုအခါ server မှ Client ၏ request ကိုလက်ခံရရှိမည်ဖြစ်သည်။ ရရှိလာသော request တွင်ပါသော email နှင့် password ကိုယူပြီး အချက်အလက်မှန်မမှန်ကို Database အတွင်းရှိ data များနှင့်တိုက်စစ်သည်။ Client မှ ထည့်ပေးလိုက်သော အချက်အလက်မှန်ပြီ ဆိုလျှင် ၎င်း user နှင့်သက်ဆိုင်သော အချက်အလက်များကို Client သို့ response ပြန်ချပေးသည်။ အချက်အလက်မှားနေလျှင် မှားနေကြောင်း Client ကို response ပြန်သည်။

About HTTP
HTTP = Hyper Text Transfer Protocol
      HTTP ဆိုသည်မှာ File များကို ကျယ်ကျယ်ပြန့်ပြန့်ဖြင့် device တစ်ခုနှင့်တစ်ခုအကြား network ပေါ်မှ transferလုပ်ပေးနိုင်သော protocol တစ်ခုဖြစ်ပါသည်။ တစ်နေရာနှင့် တစ်နေရာနီးစရာလည်း မလိုဘဲ အဝေးမှ ကျယ်ကျယ်ပြန့်ပြန့် Transfer လုပ်လို့ရသော protocol သည် HTTP ဖြစ်ပါသည်။ HTTP သည် WWW (World Wide Web) အတွက် အဓိက protocol တစ်ခုဖြစ်ပါသည်။ User နှင့် Web-Server အကြား၌ သုံးသော protocol တစ်ခုဟု အခြေခံအားဖြင့် ပြောလို့ရနိုင်ပါသည်။
     Client နှင့် Server အကြား HTTP ကိုအသုံးပြုပြီး Application ကိုရအောင်လုပ်ပါမည်။ Browser သည် HTTP client ဖြစ်ပြီး Web Server သည် HTTP server ဖြစ်ပါသည်။ Web သည် Client-Server Architecture အရ တည်ဆောက်ခြင်း ပဲ ဖြစ်ပါ တယ်။
     ထို့ကြောင့် Client သည် လိုချင်တာကို လှမ်းတောင်းပါလိမ့်မည်။ ဥပမာ Web-url တစ်ခု ကနေပြီး browser မှတစ်ဆင့် Pageတစ်ခု ကိုခေါ်ပေးပါဆိုပြီး browser ကိုလှမ်းတောင်းပါလိမ့်မည်။ ထိုသို့ လုပ်ခြင်းကို request လုပ်တယ်လို့ပြောလို့ရပါသည်။ Web client နှင့် Web Browserကြား HTTP protocol ကို သုံးပါသည်။ ဤနေရာ၌ စာရေးသူက ပိုနားလည်လွယ်အောင် Protocol အကြောင်းကို အနည်းငယ်ရှင်းပြပါမည်။
   Protocol ဆိုသည်မှာ Computer တစ်လုံးနှင့် တစ်လုံး အကြား ဆက်သွယ်နိုင်ရန်လုပ်ထား သောသဘောတူညီချက် တစ်ခုဟု ပြောလို့ရပါတယ်။ ဥပမာဆိုရလျှင် လူတစ်ယောက်နှင့် တစ်ယောက် ဆက်သွယ်ရာတွင် ဘာသာစကားကိုအသုံးပြု၍ ဆက်သွယ်ရသကဲ့သို့ Protocol သည်လည်း ထိုနည်း၄င်းပဲဖြစ်ပါတယ်။ HTTP သည် TCP(Transaction Control Protocol) ကို based  လုပ်ထားသော text based protocol ဖြစ်ပါတယ်။
    HTTP ဟုဆိုလျှင် HTTP request နှင့် HTTP response အကြောင်းသေသေချာချာ သိထားဖို့လိုအပ် ပါတယ်။ HTTP request မှာအပိုင်း၂ပိုင်းပါဝင်ပါတယ်။ First Part ကတော့Header ဖြစ်ပြီးတော့ Second Part ကတော့ Body ဖြစ်ပါတယ်။ HTTP request နှင့် response ပုံစံ အသေးစိတ်ကို စာရေးသူနောက်ပိုင်းတွင်ဆက်လက်ရှင်းပြပေးပါမည်။
       ပထမဆုံး Client သည် Server ထဲ၌ ရှိနေသော Application ကနေ data ကို HTTP requestသုံးပြီးလှမ်းတောင်းပါသည်။ လှမ်းတောင်းပြီဆိုတာနှင့် Server သည် ၎င်း request လုပ်သော data ကို ရှာပါသည်။
     HTTP protocol ကို request-response protocol လို့လည်းခေါ်ပါသေးသည်။ request -response ပုံစံကို HTTP Transaction လို့လည်းခေါ်ပါသည်။ Web client သည် ၎င်း လိုချင်တာကို request နှင့် တောင်းပါတယ်။ ထို requestကိုserver မှ ဖတ်ပါသည်။
   Serverဘက်မှအဆင်ပြေလျှင် response တစ်ခုပြန်ပေးပါလိမ့်မည်။ Requestတစ်ခုလုပ်တိုင်း လုပ်တိုင်းမှာ  response တစ်ခုပြန်လာပါမယ်။ ခြုံပြောရလျှင် HTTP ဆိုသည်မှာ user နှင့် web server အကြားရှိ Protocol တစ်ခုပဲဖြစ်ပါတယ်။ HTTP ၏ တာဝန်သည် တောင်းတာကိုပြန်ပေးရ ခြင်းပဲဖြစ်ပါတယ်။
     HTTP protocol ကို textual,Stateless protocol လို့လည်းခေါ်ပါသေးတယ်။ Client ဘက်မှ request တစ်ခုပို့လိုက်ချိန်၌ Server မှ response ကိုပြန်ပေးပါသည်။ Server သည် client မှ ဘယ် server ကနေမှတစ်ဆင့် ဘယ် data တောင်းတယ်ဆိုတာမျိုး လုံးဝမမှတ်ထားပေ။ Http protocolသည် ဘယ် Transaction ဘယ် state ကိုမှမသိမ်းထားပါပေ။ ထို့ကြောင့် server သည် client ၏ အခြေအနေကို လုံးဝမသိနိုင်ပါပေ။
      Stateless protocol ဆိုတာသည် Web Application အတွက်သိပ်အဆင်မပြေနိုင်ပေ။ အဓိက အကြောင်းသည် Web Application တွေသည် state ထိန်းဖို့ လိုအပ်ပါသည်။ ဥပမာ - login ဝင်ထားသော stateကိုထိန်းနိုင်ဖို့လိုအပ်ပါတယ်။
   ပုံမှန်အားဖြင့် state ထိန်းနည်း ၂နည်းရှိပါတယ်။ Cookie နှင့် Session ကိုသုံးလို့ရပါတယ်။ Cookie ကို browserမှာသိမ်းပါတယ်။ Session ကိုတော့ web server ပေါ်မှာသိမ်းလေ့ရှိပါတယ်။
   HTTP အပြင် HTTPS ဆိုတာလည်းရှိပါသေးတယ်။ HTTPS ဆိုတာ HTTP ကို ပို secure ဖြစ်အောင်လုပ်ထားသော Protocol ပဲဖြစ်ပါတယ်။ User နှင့် Server အကြား data တွေကိုပိုမို လုံခြုံမှု ရှိအောင်ပေါက် ကြားမှုမရှိအောင်လုပ်ပေးပါတယ်။ ကြားခံ Encryption လုပ်ပေးပြီးမှသာ Data ပို့ဆောင်ဆက်သွယ်သောအလုပ်ကိုလုပ်သောကြောင့် HTTP ထက်ပို secure ဖြစ်ပါတယ်။
     HTTP သည် Data တွေကို Encrypt မလုပ်ပါဘူး။ HTTP သည် Application Layerပေါ်မှာပဲဖြတ်သန်းပါတယ်။ ထို့ကြောင့် လုံခြုံမှုအားနည်းပါတယ်။

ထို 7မျိုး၌ ပိုပြီးလူသုံးများသော Method တွေသည် GET နှင့် POST method တွေပဲ ဖြစ်ပါတယ်။ Method အပေါ်မူတည်ပြီး Web Server ဘက်မှဘာလုပ်မလဲဆိုတာကိုပါဆုံးဖြတ်ပေးပါသည်။
     Protocol တစ်ခုမှာ သွားရမည့် Message Format သည် အတိအကျ ရှိပါတယ်။ ထို Format သည် ထို Protocol တစ်ခု၏ အသက်ပဲဖြစ်ပါတယ်။ (/)Forward slash နှင့် Space တွေမှာလည်း Meaningအသီးသီးရှိပါတယ်။ HTTP မှာ အဓိကအသက်သည် Request Format နဲ့ Response Format ဖြစ်ပါတယ်။
     GET method နှင့် တွဲသုံးမည်ဆိုလျှင် Header ထဲ၌ information only ပဲပါပါလိမ့်မည်။ POST method နှင့် တွဲသုံးမည်ဆိုလျှင်တော့ body ထဲ၌ပါ information ပါလာပါလိမ့်မည်။

Request
An example HTTP request

HTTP request သည် Header မှာ ပုံတွင်ပြထားသည့်အတိုင်း Method ပါလာရပါမည်။ Method ဆိုတာ သည် client လုပ်ချင်သော operation type အမျိုးအစားဖြစ်ပါသည်။
    ဒုတိယတစ်ခုအနေနဲ့ Url or Path ပါရပါမည်။ နောက်ဆုံးအပိုင်းအနေနဲ့ Version of the protocol ပါဝင်ရပါမယ်။ Host သည် web browser name ကိုဆိုလိုခြင်းဖြစ်ပါသည်။ GET method မှာပါသော request body သည် 64KB မျှသာ သိမ်းနိုင်သောကြောင့် File Upload တွင် POST Method ကိုသာသုံးသင့်ပါသည်။

Response
An example HTTP response

Response ၏ Header ထဲ၌လည်း Protocol ၏ version ပါပါလိမ့်မည်။ request အောင်မြင်စွာရောက်သွားလားမရောက်သွားလားဆိုတာကိုပြသော status code ပါပါလိမ့်မည်။ Status Code 200 ဆိုသည်မှာ request operation သည် OK သွားကြောင်းဆိုလိုခြင်းဖြစ်ပါသည်။ Status Code 500 ဆိုသည်မှာ Server ၌ Error တစ်ခုခုတက်သွားကြောင်း ဆိုလိုခြင်း ဖြစ်ပါသည်။ Status Message ပါပါမည်။ Content-type ဆိုတာကတော့ browser ကိုပြန်လာတဲ့ ပုံစံဖြစ်ပါသည်။

ဒါကိုသေချာကြည့်ချင်လျှင် Chrome မှာဖြစ်စေ မိမိကြိုက်နှစ်သက်ရာ browser မှာ ဖြစ်စေ Right Click ထောက် Inspect ကိုနှိပ်ပြီး request တစ်ခုတောင်းကြည့်လျှင် Network option ထဲမှာတက်လာသော request နှင့် response တွေကနေတစ်ဆင့်ကြည့်လို့ရပါတယ်။
