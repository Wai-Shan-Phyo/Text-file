Data Structure P3

Data Structure (Part - 3)
====================

Structured Type
----------------------------
Structured type ဆိုတာ Basic data structure တွေနဲ့ defined လုပ်ထားတဲ့ data တွေကို စုစည်းထားတဲ့  Data Structure တစ်ခုပါ။ Structured type ကို array type နဲ့ record type ဆိုပြီး 2 မျိုးခွဲထားပါတယ်။

‌Array Type
------------------
Array ဆိုတာ Table ကိုခေါ်တာပါ။ Array မှာပါတဲ့ data တွေဟာ type နဲ့ size တွေတူရပါမယ်။ ဆိုလိုချင်တာ same type, same size ပါ။ ဥပမာ -

[1, 2, 3, 4, 5]
[
    ["Apple", "Red"],
    ["Orange", "Orange"],
    ["Banana", "Yellow"],
]

Array ထဲမှာပါတဲ့ data item တွေကို array element ဒါမှမဟုတ် table element လို့လဲ ခေါ်ပါတယ်။ Array တွေကို ဘယ်လို ဖော်ပြမလဲ၊ Array ထဲမှာ data တွေကို ဘယ်လိုစီစဉ်မလဲဆိုတာကတော့ ကိုယ်သုံးတဲ့ Programming Language အပေါ်မူတည်ပါတယ်။

One Dimensional Array
---------------------------------------
One Dimensional Array ဆိုတာ Data တွေကို မျဉ်းကြောင်းပုံစံ စီစဉ်တဲ့ Data Structure ပါ။ ဥပမာ -

[1, 2, 3, 4, 5]
["One", "Two", "Three", "Four", "Five"]

Array Element တစ်ခုစီမှာ သူ့ကို indicate လုပ်မဲ့ number တစ်ခုစီရှိပါတယ်။ အဲ့ကောင်ကို Index လို့ခေါ်ပါတယ်။ များသောအားဖြင့် index ကို 0 (Zero) က စတွက်ကြပါတယ်။ index ဟာ array element တစ်ခုစီကို indicate လုပ်သလို ကိုယ်လိုချင်တဲ့ array element data ကို ခေါ်ချင်ရင်လဲ index ကိုအသုံးပြုလို့ရပါတယ်။ ဥပမာ -

A = ["red", "green", "blue"]
A[0] = "red"
A[1] = "green"
A[2] = "blue"

Two Dimensional Array
---------------------------------------
Two Dimensional Array ဆိုတာကတော့ data ကို table တစ်ခုလို စီချလိုက်တာပါ။ One Dimensional Array တုန်းက Horizontal Axis (x) ၀င်ရိုး တစ်ခုတည်းဆိုရင် ဒီ Two Dimensional Array မှာ Horizontal နဲ့ Vertical Axis (x, y) ၀င်ရိုးနှစ်ခု ပါ၀င်တယ်လို့ မှတ်ယူလိုက်ပါ။ Horizontal Axis (x-axis)  ကို row လို့ခေါ်ပြီးတော့ Vertical Axis (y-axis) ကို column လို့ခေါ်ပါတယ်။ နမူနာအားဖြင့် Microsoft Excel ဟာ Two Dimensional Array ပါပဲ။ ဥပမာအားဖြင့်

A = [["Apple", "Red"], ["Orange", "Orange"], ["Banana", "Yellow"]]
(Expand)
A = [
    ["Apple", "Red"],
    ["Orange", "Orange"],
    ["Banana", "Yellow"],
]

Index ရေးရင်တော့ row index ကို အရင်ရေးပြီးနောက်မှ column index ကိုရေးပါတယ် (x, y)။ Index အနေနဲ့ ဖော်ပြရရင်

A = [
    [(0,0), (0,1)],
    [(1,0), (1,1)],
    [(2,0), (2,1)],
]

အဲ့တော့ Data တွေကို Index နဲ့ခေါ်မယ်ဆိုရင် A(0,1)   သည် Red ဖြစ်လိမ့်မယ်။ A(2,0) သည် Banana ဖြစ်လိမ့်မယ်။ ဒီလောက်ဆို Two Dimensional Array အကြောင်း နားလည်မယ်လို့ ယူဆပါတယ်။

Two Dimensional Array ကို Main Memory Unit မှာ သိမ်းတဲ့အခါကျ One Dimensional Array Form အနေနဲ့ပဲ သိမ်းပါတယ်။ အဲ့လို form အနေနဲ့ သိမ်းတဲ့နေရာမှာ Row direction နဲ့စီပြီး သိမ်းတာရှိသလို Column direction နဲ့စီပြီး သိမ်းတာလဲ ရှိပါတယ်။ ဥပမာ အပေါ်က array ကို row direction အနေနဲ့ သိမ်းမယ်ဆိုရင် (0, 0), (0, 1), (1, 0), (1, 1), (2, 1), (2, 2) ဆိုပြီး သိမ်းပါမယ်။ column direction အနေနဲ့ဆိုရင်တော့ (0, 0), (1, 0), (2, 0), (0, 1), (1, 1), (2, 1) ဆိုပြီး ဖြစ်သွားပါမယ်။ ဘယ်လို direction နဲ့ သိမ်းမလဲဆိုတာကတော့ ကိုယ်သုံးတဲ့ Programming Language ရဲ့ Compiler အပေါ်မူတည်ပြီး ကွဲပြားသွားပါမယ်။ ဥပမာ Fortran မှာ Column Direction အနေနဲ့ သိမ်းပြီး COBOL မှာတော့ Row Direction အနေနဲ့ သိမ်းပါတယ်။

Multi-Dimensional Array
----------------------------------------
Two ထက်ပိုတဲ့ array type တွေကတော့ multi-dimensional array ဖြစ်သွားပါလိမ့်မယ်။ ဥပမာ - three, four, five, six, etc...
ဘယ်လောက်ပဲ multi-dimensional ဖြစ်ဖြစ် programming language နဲ့ compiler အပေါ်မူတည်ပြီး dimension ကို define လုပ်မဲ့ limitations number ကတော့ ရှိပါတယ်။

Record Type
----------------------
Structured type ရဲ့ ဒုတိယ type ဖြစ်တဲ့ record type ကတော့ Array type လိုပါပဲ။ Array နဲ့မတူတာကတော့ data types အမျိုးမျိုးကို သိမ်းလို့ရပြီး Data တွေရဲ့ Index ကို နာမည်ပေးရတာပါပဲ။ ဥပမာ -

[
    {
            "Student_ID": 001,
             "Name": "Mg Mg"
             "Marks": 67
     },
     {
            "Student_ID": 002,
             "Name": "Hla Hla"
             "Marks": 58
     },
     {
            "Student_ID": 003,
             "Name": "Aung Aung"
             "Marks": 85
     },
]

Abstract Data Type
---------------------------------
Basic Data Structure ရဲ့ သုံးခုမြောက် data structure ပါ။ OOP (Object-oriented Programming) သမားတွေဆို သိပါတယ်။ Abstraction ရဲ့ သဘောက နောက်ကွယ်က operation or process ကို hidden လုပ်ထားပြီး လိုချင်တဲ့ result ကိုပြန်ထုတ်ပေးတာပါ။ အနောက်က process ကို ဂရုစိုက်စရာမလိုပဲ အလိုရှိသလို ခေါ်သုံးနိုင်ပါတယ်။ ဥပမာ ကျွန်တော်တို့ phone operator တွေ ဘယ်လိုအလုပ်လုပ်လဲ သိစရာမလိုပဲ ဖုန်းပြောကြတာမျိုးပေါ့။ အဲ့လို data structure မျိုးကို abstract data type လို့ခေါ်ပါတယ်။
