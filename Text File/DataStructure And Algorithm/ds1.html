Data Structure (Part -1)
====================
Data Structure တွေအကြောင်းကိုမပြောခင် Data နဲ့ပတ်သက်တာလေးတွေကို အရင်ပြောပြသွားပါမယ်။

အရင်ဆုံး Data ဆိုတာ ဘာလဲပေါ့။ Data ဆိုတာ values ဒါမှမဟုတ် values အစုအဝေး ကို ခေါ်ပါတယ်။ ဥပမာ - Employee Data ဆိုရင် Employee နဲ့ ပတ်သက်တဲ့ Value အစုအဝေး (Name, ID, Address, etc...) တွေပါ၀င်ပါတယ်။

အဲ့လို Values အစုအဖွဲ့ထဲကမှ Value တစ်ခုကို Data Item လို့ခေါ်ပါတယ်။ ဥပမာ - Employee Data ထဲက ခုနက ပြောသွားတဲ့ Name, ID, Address  စတာတွေပေါ့။ အဲ့တာတွေဟာ Data Item တွေပါ။

အဲ့ဒီ့ Data Item ကို Group Items လို့ခေါ်တဲ့ subitems လေးတွေအဖြစ် ထပ်ခွဲလို့ရပါတယ်။ ဥပမာ - Employee Name ဆိုတဲ့ Data Item တစ်ခုကို First Name, Middle Name, Last Name ဆိုတဲ့ Group Item လေးတွေအဖြစ်ထပ်ခွဲလို့ရပါတယ်။

Computer ကနေ processed လုပ်ခံရတဲ့ တူညီတဲ့ Data အစုအဝေးတွေကို Data Type လို့ခေါ်ပါတယ်။
ဥပမာ - Employee Name ဆိုရင် Character, Phone No ဆိုရင် Integer စသဖြင့်ပေါ့။

Program Design Stage မှာတော့ Computer ထဲမှာ Processed လုပ်ရမဲ့ data တွေကို စစ်ဆေးရပါတယ်။ အဲ့သလိုစစ်ဆေးမှလဲ ဘယ် Data ဟာ ဘယ် data type နဲ့ ကိုက်ညီသလဲဆိုတာကို ရွေးချယ်ပေးနိုင်မှာပါ။ ဥပမာ - Name ထဲမှာ ဝင်လာမဲ့ Data ကိုစစ်ဆေးပြီးတော့ ဒီ Data type ဟာ Character ဖြစ်နိုင်မလား၊ Integer ဖြစ်နိုင်မလား၊ String ဖြစ်နိုင်မလားဆိုတာကို ဆုံးဖြတ်နိုင်ပါတယ်။

အပေါ်မှာ ပြောသွားတဲ့ Data စုဖွဲ့ပုံတွေ၊ တည်ဆောက်ပုံတွေ သာမက Data အလုပ်လုပ်ပုံတွေ၊ သိမ်းဆည်းပုံတွေ၊ စစ်ဆေးပုံတွေ၊ ဆက်နွယ်ပုံတွေကို လေ့လာပြီး လက်တွေ့အသုံးချရတဲ့ အရာကို Data Structure လို့ခေါ်ပါတယ်။

အောက်က ပုံကတော့ Data Structure မှာပါ၀င်တဲ့ Data Tဝype တွေကို classify လုပ်ပြထားတဲ့ပုံပါ။ နောက်အပိုင်းတွေမှာ တစ်ခုစီကို ရှင်းပြသွားပါမယ််
