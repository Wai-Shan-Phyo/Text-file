Inheritance ဆိုတာက ဘယ်လိုအရာမျိုးလဲ?

Inheritance (အမွေဆက်ခံခြင်း)
   အားလုံးပဲ OOPဆိုတဲ့ Object-oriented Programming ကိုတော့ ရင်းနှီးနေကြမှာပါ။ Inheritance ဆိုတာကလည်း OOP ထဲက အရေးကြီးတဲ့ အခြေခံအုတ်မြစ်တွေထဲက တစ်ခုအပါအဝင်ပဲ ဖြစ်ပါတယ်။ သူက Class တစ်ခုကနေ တခြား Class ရဲ့ property တွေကို ရယူခြင်း (ဒါမှမဟုတ်)  မှီခိုခြင်းဖြစ်တယ်။
ဥပမာအားဖြင့် ကလေးတစ်ယောက်ဟာ သူရဲ့ မိဘတွေဆီက စရိုက်ကို အမွေဆက်ခံလိုက်တာလိုမျိုးပါပဲ။

Inherit လုပ်တဲ့ Class ကိုတော့ Sub Class ဒါမှမဟုတ် Derived class, Child class လို့ ခေါ်ပြီးတော့ Properties တွေ Inherited ဖြစ်တဲ့ Class ကိုတော့ Super class (Base Class, Parent Class ) လို့ ခေါ်ကြပါတယ်။

    Inheritance ကို အသုံးပြုခြင်းနဲ့ အိုင်တို့က တည်ဆောက်ထားပြီးသား ရှိပြီးသားဖြစ်တဲ့ Class ထဲက Fields တွေ Methods တွေကို ပြန်ခေါ်ပြီးတော့ အသုံးပြုလို့ရပါတယ်။  အဲ့ဒါကြောင့် Inheritance facilitates Reusability and အဲ့ဒါက OOP ထဲမှာ အရေးကြီးတဲ့ Concept တစ်ခုပဲ ဖြစ်ပါတယ်။

Different Types of Inheritance

1. Single Inheritance
2. Multi-level Inheritance
3. Multiple Inheritance
4. Hierarchical Inheritance
5. Hybrid Inheritance
6. Multipath Inheritance  ဆိုပြီးတော့ 6 မျိုးကို OOP က support ပေးပါတယ်။ ဒါပေမယ့် No. 6 Multipath Inheritance ကို Java မှာ Support မလုပ်ထားတဲ့အတွက် အိုင်တို့ သုံးလို့ မရပါဘူး​။

1. Single Inheritance -
        Sub-class တွေကနေပြီးတော့ Superclass တစ်ခုရဲ့ features တွေကို အိုင်တို့ ယူသုံးလို့ ရပါတယ်။
Example ဆိုရရင် Sub Class B extends Super Class A ပါ။

//Code

class one {

    public void print_geek()
    {
        System.out.println("Geeks");
    }
    }

class two extends one {

    public void print_for() { System.out.println("for"); }
    }
    // Derived class
   
public class Main {

    public static void main(String[] args)
    {
        two g = new two();
        g.print_geek();
        g.print_for();
        g.print_geek();
    }
    }

//output
Geeks
for
Geeks//

2. Multi-level Inheritance -
                 Multi-level Inheritance မှာဆိုရင် Derived class တစ်ခုက Base class ကနေပြီး Inherit လုပ်နိုင်ပြီး အဲ့ Derived class ကနေပဲ တခြား Derived class နောက်တစ်ခုအတွက် Base class အနေနဲ့ ဆောင်ရွက်နိုင်ပါတယ်။
ဥပမာပြောရရင် အဖိုးဆီက အဖေက အမွေဆက်ခံလာပြီး အဖေ့ဆီက အမွေကို သားက ထပ်ဆက်ခံသလိုမျိုးပါ။

//code

class one {

    public void print_geek()
    {
        System.out.println("Geeks");
    }
    }

class two extends one {

    public void print_for() { System.out.println("for"); }
    }

class three extends two {

    public void print_geek()
    {
        System.out.println("Geeks");
    }
    }
// Derived class

public class Main {

    public static void main(String[] args)
    {
        three g = new three();
        g.print_geek();
        g.print_for();
        g.print_geek();
    }
    }

//output
Geeks
for
Geeks//

3. Multiple Inheritance(Through Interface)
           Class တစ်ခုမှာ Base class တွေ အများကြီးရှိနိုင်ပြီးတော့ Parent class အကုန်လုံးဆီကနေ features တွေကို Inherit လုပ်လို့ ရတာမျိုးပါ။
ဥပမာအ​နေနဲ့က​တော့ ကလေးတစ်ယောက်က အဖေ့ဆီက အမွေရော အမေ့ဆီက အမွေရော အဒေါ်ဆီက အမွေကိုရော ဆက်ခံနိုင်ပြီး သုံးလို့ရတာမျိုးပါနော်။
ဒါပေမယ့် အိုင်တို့တစ်ချက်သတိထားရမှာက Java က class အများကြီးနဲ့ Multiple Inheritance ကို သုံးခွင့်မပေးပါဘူး။ ဒါကြောင့် Java မှာဆိုရင် အိုင်တို့က Interfaces တွေကို အသုံးပြုပြီးပဲ ဒီ Multiple Inheritance တွေကို ရေးလို့ရပါတယ်။

//code

interface one {

    public void print_geek();
    }

interface two {

    public void print_for();
    }

interface three extends one, two {

    public void print_geek();
    }
class child implements three {

    @Override public void print_geek()
    {
        System.out.println("Geeks");
    }

    public void print_for() { System.out.println("for"); }
    }
// Drived class

public class Main {

    public static void main(String[] args)
    {
        child c = new child();
        c.print_geek();
        c.print_for();
        c.print_geek();
    }
    }

//output
Geeks
for
Geeks//
4. Hierarchical Inheritance
             Class တစ်ခုက တစ်ခုထက်ပိုတဲ့  Sub class​တွေအတွက်  Base class အဖြစ် လုပ်ဆောင်ပြီး သူ့ရဲ့ Features တွေကို extends လုပ်ထားတဲ့ sub class တွေက ယူသုံးလို့ ရပါတယ်။
ဥပမာ သားသုံးယောက်က အဖေတစ်ယောက်ထဲကဆီက အမွေဆက်ခံခြင်း။

//code

class A {

    public void print_A() { System.out.println("Class A"); }
    }

class B extends A {

    public void print_B() { System.out.println("Class B"); }
    }

class C extends A {

    public void print_C() { System.out.println("Class C"); }
    }

class D extends A {

    public void print_D() { System.out.println("Class D"); }
    }
// Driver Class

public class Test {

    public static void main(String[] args)
    {
        B obj_B = new B();
        obj_B.print_A();
        obj_B.print_B();

        C obj_C = new C();
        obj_C.print_A();
        obj_C.print_C();

        D obj_D = new D();
        obj_D.print_A();
        obj_D.print_D();
    }
    }

//output
Class A
Class B
Class A
Class C
Class A
Class D//

5.Hybrid Inheritance (Through Interfaces)
             Hybrid Inheritance က အပေါ်က နှစ်ခု ဒါမှမဟုတ် နှစ်ခုထက်ပိုတဲ့ Inheritance types တွေကို ရောပြီး အသုံးပြုရပါတယ်။ ဒါပေမယ့် အဲ့လို class တွေအများကြီး extends လုပ်တာကို Java က လက်မခံတဲ့အတွက်ကြောင့် အိုင်တို့က သူ့ကိုသုံးချင်ရင် Interface နဲ့ အသုံးပြုရပါတယ်။

•Default Superclass: Object class မှ အပ၊ Superclass မရှိ၊ class တိုင်းမှာ one and only one(single inheritance) Superclass ပဲ ရှိပါတယ်။ တခြား explicit superclass တွေ မရှိတဲ့အခါမှာ class တိုင်းဟာ Object class ရဲ့ sub class တွေ ဖြစ်တယ်။

•Superclass can only be one: Superclass တစ်ခုက Sub class တွေအများကြီးရှိလို့ ရပေမယ့် sub class ကတော့ superclass တစ်ခုပဲ expends လုပ်လို့ရပါတယ်။ Java က Multiple Inheritances with classes ကို Support မလုပ်ပါဘူး။ Interface တွေကို သုံးပြီးတော့ပဲ Java မှာ Multiple inheritance ကို ရေးလို့ရနိုင်ပါတယ်။

•Inheriting Constructor: Sub class တစ်ခုက သူ inherit လုပ်ထားတဲ့ Superclass ရဲ့ members တွေ ဖြစ်တဲ့ (fields, methods, nested classes) တွေအကုန်လုံးကို ယူပြီး သုံးလို့ ရပါတယ်။ constructor တွေက member မဟုတ်ကြတဲ့အတွက် အိုင်တို့က constructor တွေကိုတော့ ယူသုံးလို့ မရဘူးပေါ့နော်။ ဒါပေမယ့် Superclass ရဲ့ constructor တွေကို Sub class ကနေပြီးတော့ invoke လုပ်လို့ ရပါတယ်။

•Private Member Inheritance: Sub class တစ်ခုက သူ့ရဲ့ parent class က private နဲ့ သုံးထားတဲ့ member တွေကို inherit လုပ်လို့ မရပါ။ Super class က​နေ သူ့ရဲ့ private fields က ကြေငြာတဲ့ public or protected methods ( getter, setter) method တွေကိုတော့ Sub class က ပြန်ပြီး သုံးလို့ ရပါတယ်။

What all can be done in a Subclass?

Sub class ထဲမှာဆိုရင် အိုင်တို့ဟာ replace လုပ်လို့ရမယ် ၊ hide လို့ရမယ် ၊ member အသစ်​တွေကို supplement လုပ်လို့ရမယ်။

•Inherited fields တွေကို တခြား fields တွေလိုမျိုးပဲ တိုက်ရိုက်ယူပြီးအသုံးပြုလို့ ရပါတယ်။

•အိုင်တို့ဟာ Sub class ထဲမှာ Superclass ကမဟုတ်တဲ့ fields အသစ်တွေကိုလည်း declare လုပ်နိုင်ပါတယ်။

•Inherited method တွေကိုလည်း အဲ့အတိုင်းပဲ Sub class ထဲမှာ တိုက်ရိုက်သုံးလို့ရပါတယ်။

•Superclass ထဲမှာ ရှိတဲ့ instance method နဲ့ signature တူတဲ့ instance method အသစ်တစ်ခုကို Sub class ထဲမှာ ရေးနိုင်ပြီးတော့ အဲ့ဒါကိုတော့ Overridding လို့ ခေါ်ပါတယ်။

•Superclass မှာရေးခဲ့တဲ့ static method နဲ့ signature တူတဲ့ အသစ်တစ်ခုကို Sub class ထဲမှာ ထပ်မံ ရေးလို့ရပြီး အဲ့ဒါကတော့ hiding လုပ်လိုက်တယ်လို့ ခေါ်ကြပါတယ်။

•Superclass က မဟုတ်တဲ့ method အသစ်တွေကိုလည်း sub class ထဲမယ် declare လုပ်နိုင်ပါတယ်။

•Super class က instructor ကို ဖော်ပြနိုင်တဲ့ sub class constructor ကိုလည်း ရေးနိုင်ပါတယ်။

Advantages of Inheritance

1. Code များ မလိုအပ်ဘဲ ပိုနေမှုကို လျှော့ချပေးခြင်း။
2. Code ပြန်လည်အသုံးချနိုင်ရန် စီစဉ်ပေးခြင်း။
3. Source code size ကို လျှော့ချပေးပြီး code readability ကို တိုးမြှင့်ပေးနိုင်ခြင်း။
4. Code တွေက စီစဉ်ဖို့ အရမ်းလွယ်ကူပြီး Parent class နဲ့ Child class ဆိုပြီး ခွဲခြားနိုင်ခြင်း။
5. Base class နဲ့Child classes တွေထဲမှာ Overridding ဖြစ်လာတဲ့ code extensibility ကိုလည်း Support ပေးနိုင်ခြင်း။
