ကျောင်းသားတွေအနေနဲ.သင်သာ သင်နေရတယ် ကိုနဲ.မဆိုင်ဘူးထင်တယ် ဆိုပြီး ပျင်းရိ ငြီးငွေ.စိတ်ပျက် (အဆဲခံရတော့မယ် ဟေ့ အဆဲခံရတော့မယ်) တာမျိုးကြုံဖူးကြမှာပါ။

ဒီလိုဖြစ်ရတဲ့ အရင်းခံက ဒါတွေက သင်သာသင်နေရတယ် ဘယ်မှာသုံးလို.သုံးမှန်းမသိဘူးဆိုတဲ့ အတွေးတွေကနေစတာပါ။ တန်ဖိုးရှိမှန်းမသိဘူး အသုံး၀င်မှန်းလဲမသိဘူး  ဒါဆို စိတ်၀င်စားဖို.လုံး၀ ကိုမဖြစ်နိုင်တော့ပါဘူး။

အဲ့တော့ စိတ်၀င်စားဖို.ဆိုတာက ဒါကြီးကဘယ်မှာသုံးတယ် ဘယ်နေရာအတွက်တော့ အသုံး၀င်တယ် ဒါတွေမသိရင် ဘာတွေမသိနိုင်တော့ဘူးဆိုတာကို ရှင်းပြဖို.လိုပါတယ် (ဒါသက်သက်ကျပ်တာ :P)

Algorithm and Data Structure
ဒီကောင်တွေကတော့ programming မှာတိုက်ရိုက်သုံးပါတယ် Algorithm ,DatatStructure ရယ် Big O notation ကိုကောင်းကောင်းမသိဘူးဆိုရင် ဘယ် Algorithm ကပိုမြန်မယ် နှေးမယ်ဆိုတာကိုမသိပါဘူး ။နောက်ဘယ်အချိန်မှာ ဘယ်လို problem မျိုးကို ဘယ်လို Algorithm မျိုးသုံးပြီးရှင်းရင်ရနိုင်တယ်ဆိုတာမသိပါဘူး။ ဥပမာ data ကိုခနခန access လုပ်တဲ့ data structure ဆိုရင် Hash ကိုသုံးမှာလား array လို linkedlist လိုကောင်မျိုးသုံးရမှာလားကို ခွဲခြားနိုင်ပါလိမ့်မယ်။ (Access လုပ်ဖို.အတွက်ဆို Hash ကပိုမြန်တယ်။)

DBMS Relational Calculus.Tuple Calculus
DBMS မှာ relational calculus , tuple calculus ဆိုတာတွေကို စိတ်ညစ်အောင်တွက်ရဘူးမှာပါ။ ဒီ relatoinal calculus , tuple calculus တွေကို ဘယ်နေရာမှာသုံးလဲဆိုရင်တော့ DMBS implement လုပ်တဲ့အခါ(ဆိုချင်တာက ကိုယ်တိုင် MySQL လို DBMS တခုရေးကြည့်တဲ့အခါလိုပါတယ်)။ လွယ်လွယ်ပြောရရင် Assembly ဆိုတာ programming language တွေအတွက် ဟိုးအောက်ဆုံး သွားရတဲ့ language ဖြစ်သလို။ Relational Calculus , tupe calculus တွေဟာလဲ SQL အတွက် Assembly လို.ပဲဆိုရမှာပါ။ ဒါတွေကိုသေချာနားလည်တော့မှ Join Query optimization, query execution plan, quest cost အစရှိတာတွေကိုတွက်နိုင်မှာပါ။ ဒါဆိုရင် ကိုက SQL ရေးတဲ့အခါမှာ JOIN ပြီးမှ Select လုပ်တာ ကောင်းသလား Select အရင်လုပ်ပြီးမှ JOIN တာကောင်းသလား ဘယ်ဟာက ပိုမြန်မလဲဆိုတာကို သိပါလိမ့်မယ်။ DBMS အထဲမှာ set theory ကိုလဲ internally မှာသုံးပါသေးတယ်။

Math
Algorithm တွေအစကသချာင်္ကလို.ဆိုရမှာပါ။ GCD (Greatest Common Divisor) ဆိုတာ Euclid ရေးခဲ့တဲ့ Algorithm ပါ။ Math ကို discrete math နဲ. Calculus ဆိုထပ်ခွဲပါမယ်။ Discrete math မှာသင်ရတဲ့ set, number , matrix အစရှိတာတွေကို machine learning algorithm တော်တော်များများမှာ သုံးပါတယ်။ ဥပမာ SVM ဆိုရင် Vector အကြောင်းကိုတော်တော်နားလည်ရပါလိ့မ်မယ်။ Bayesian algorithm ဆိုရင် probability ကိုအခြေခံပြီးတွက်တာပါ။ function ဆိုရင် functional programming တွေမှာ မသိမဖြစ်ပါ။ Trigo တွေ ဆိုရင်တော့ Computer Graphic , Image processing တွေမှာသုံးပါတယ်။ Matrices တွေဘယ်လိုမြှောက်ရမယ်ဆိုတာမသိရင် CG 2D,3D game ရေးဖို. မစဉ်းစားပါနဲ.တော့။ Set, Category Theory, Function ဒါတွေမသိရင် functional programming ကိုခြေခြေမြစ်မြစ်နားလည်ဖို.မလွယ်ဘူး။

Calculus ကတော့ Image processing တွေ wave နဲ.သိမ်းတဲ့ sound , video processing တွေမှာ သုံးပါတယ်။

Physic
ရူပဗေဒကိုနားမလည်ရင် Game programming လိုကောင်မျိုးမှာ သဘာ၀ကျကျ သဘာ၀ဆန်တဲ့ ဥပမာဗျာ ရေလှိုင်းလေးစီးနေတာမျိုး effect မျိုးဟာ physic law တွေ အပေါ်မူတည်ပြီးလုပ်ထားတာပါ။ ကိုယ်တိုင်လုပ်ကြည့်မယ်ဆိုရင်တော့ Physic မသိရင် လစ်ပါလို.ပြောရမှာပါ။

Compiler ,Programming Language

OS
ကျောင်းသားတိုင်းတော့ OS ရေးမှာမဟုတ်ပါဘူး ဒါပေမဲ့ OS မှာပါတဲ့ Thread, Deadlock, စတာတွေကိုသိမှ concurrent programming ကိုသေသေချာချာနားလည်မှာပါ။ နောက် OS ရဲ. file management ကိုနားလည်မယ် CO အကြောင်းလဲသိမယ်ဆိုရင် data ကို file ကဖတ်တာပိုမြန်လား memory ကဖတ်တာ ပိုမြန်မလားဆိုတာကိုသိပါလိမ့်မယ်။ ဘာလို. Buffer ခံပြီးဖတ်ရင် ပိုမြန်တာလဲ။ Node.js က Asynchronous non-blocking ဆိုတာ ဘာကြောင့်ပိုမြန်တာလဲဆိုတာကို လွယ်လွယ်ကူကူနားလည်နိုင်မှာပါ။.
