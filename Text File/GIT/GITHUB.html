What is Git

      Git က English Definition လိုဆိုရင်​တော့ version control system ပါ။ ကျွန်​တော်တို့နားလည်​အောင်​ပြောရမယ်ဆိုရင် file ​တွေမှာကျွန်​တော်တို့ code ​တွေ​ရေးလိုက်တိုင်း code ​တွေကို track လိုက်ပြီး ဘာ code ​တွေအသစ်ထည့်လိုက်တယ်။ ဘယ် file ​တွေကိုပြင်လိုက်တယ်။ ဘယ် code ​တွေကိုဖျက်လိုက်တယ်ဆိုတာ​တွေကို version ​တွေအလိုက် commit လုပ်လိုက်တိုင်း သိမ်းပေးထားတယ်ဗျ။ အဲ့လိုဆို​တော့လိုရင်ပြန်ကြည့်ရင်ပဲဖြစ်ဖြစ် ပြန် check ချင်တာပဲဖြစ်ဖြစ် version ၂ခုကို compare လုပ်ချင်တာပဲဖြစ်ဖြစ်လုပ်လို့ရတယ်။တခြားdeveloper ​တွေနဲ့ project အတူ​ရေးရင်လည်း သူတို့ဘာ​တွေ ​​ရေးလိုက်တယ် / ဘာ​​တွေပြင်လိုက်တယ် / ဘယ် file ​တွေမှာ​ရေးလိုက်တယ် / ဘယ်အချိန်က​ရေးလိုက်တယ် အဲ့လိုဟာ​တွေရဲ့အ​သေးစိတ်​ history တွေကို track လိုက်လို့ရ​​အောင်လုပ်​ပေးထားတယ်။
     
      Git က ကိုယ့်ရဲ့ code​တွေကို​တော့ local machine မှာပဲ track လုပ်ပြီး သိမ်းထား​ပေးတယ်။ ဒါ​ပေမယ့် online ပေါ်မှာတင်ပြီး တခြားသူ​တွေနဲ့ အ​ပေါ်ကလို collaborate လုပ်ချင်ရင် online hosts တွေ (github, bitbucket, gitlab) ကိုအသုံးပြုလို့ရတယ်။ ပြီး​တော့ file တစ် file ထဲမှာ developer ၂​ယောက် code ပြင်လိုက်တယ်ဆိုလဲ (conflict) ​တွေမဖြစ်​အောင် comparison တွေပြ​ပေးပြီး  သူ့ code နဲ့ကိုယ့် code ​၂ခုလုံးပါ​အောင် ပြန် modify လုပ်လို့ရတယ်။ အဲ့တာတင် မဟုတ်ဘူး သူများ open source project (public- project)တွေ ကိုလည်း terminal မှာ 'git clone "project_url" ' ဆိုတဲ့ command နဲ့ မိမိရဲ့ local machine ထဲကို download လုပ် လို့ရပါတယ်။

(Git အသုံးပြုနည်း)

     Git ကို အသုံးပြုမယ်ဆို ပထမဦးဆုံး website https://git-scm.com/downloads မှာ down ပြီး install လုပ်ရမယ်ဗျ။ (install ပြီးရင် git ကိုယ့်ရဲ့စက်ထဲကို ​ရောက်မ​ရောက် terminal မှ 'git --version' command  ဖြင့်စစ်လို့ရပါတယ်။ (optional) default terminal ကိုသုံးလို့ရ​ပေမယ့် git download ရင်ပါ​​​လာတဲ့ git bash shell ကိုသုံးချင်ဆိုရင် (VS code မှာဆို settings.json ထဲဝင်ပြီး "terminal.integrated.shell.windows": "C:\\Program Files\\Git\\bin\\bash.exe" ကို copy paste လုပ်ရင် VS Code ရဲ့ default terminal က git bash ​ပြောင်းသွားမှာပါ)။ ကိုယ့် project ကို gitနဲ့ချိတ်ချင်တယ်ဆိုရင်အရင်ဆုံး project folder ထဲဝင်ပြီး IDE (VS Code) မှာ ctrl + ` နှိပ် (integrated terminal keyboard shortcut), ပြီးရင် 'git init' လို့ရိုက်လိုက်ရင် repository (repository ဆိုတာ project folder / project ထဲက code ​တွေ, version အကုန်သိမ်းထားတဲ့ main data structure) ကို local machine မှာအသစ် လုပ်လိုက်ပြီး git နဲ့ကိုယ့် project နဲ့ချိတ်သွားပါလိမ့်မယ်။

(What is github)

     Github ဆိုတာက cloud based hosting service တစ်ခုပါ။ ကိုယ့် ရဲ့ repository ​တွေကို cloud ​ပေါ်မှာ hosting လုပ်နိုင်​​အောင်ပြုလုပ်​ပေးတဲ့ service ​ပေါ့။ သိထားရမှာတစ်ခုက git နဲ့ github ကမတူပါဘူး။ git က versional control system ဖြစ်ပီး github ကတော့ hosting service ပါ။ ဘယ်သူမဆို github account ဖွင့် repo လုပ်ပြီးကိုယ့် project code ​တွေကိုတင် github ကိုသုံးပြီး cloud ​ပေါ်မှာ free hosting တင်လို့ရပါတယ်။ Github ရဲ့ user friendly interface ​တွေ​ကြောင့် github က တခြားservice ​တွေ (bitbucket, gitlab) တို့ထက်စာရင် popular အဖြစ်ဆုံး service ဖြစ်​နေတာပါ။ Github မှာသူများ​တွေရဲ့ repository (open source project) ​တွေကိုလည်း star ​ပေးနိုင်ပြီး ကိုယ့် account ထဲမှာ copy ယူပြီးသိမ်းထားချင်ရင်လည်း fork (share) နိုင်တဲ့ feature ပါပါတယ်။

(Github မှာ repository လုပ်ပြီး ကိုယ့် code ​တွေတင်နည်း)

     Github ​ပေါ်မှာ code တင်ချင်ရင်​တော့ပထမဆုံးလုပ်ရမှာက Github website https://github.com/ မှာ account လုပ်၊ ပြီးရင် repository  အသစ်တစ်ခုလုပ်ရမှာပါ။ repository လုပ်ရင် public နဲ့ private ဆိုပြီး၂ခုရှိပါတယ်။ public ကကိုယ့် project ကိုသူများ​တွေကြည့်လို့ fork လို့ရပြီး private က​တော့ကိုယ်ကလွဲရင်ဘယ်သူမှ ကိုယ့် project ကိုကြည့်လို့မရပါ။ Developers ​တွေက​တော့ပုံမှန် project ​တွေဆို public repo နဲ့ပဲလုပ်ကြပါတယ်)။ ပြီးရင်​အောက်မှ Add a README file ဆိုတာကို tick လုပ်ပြီး create repository လုပ်ပါ။ Repo လုပ်ပြီးသွား​ပေမယ့်ကိုယ့် project version ​တွေ code ​တွေကအခု local ထဲမှာပဲရှိ​သေး​တော့ ကိုယ့်ရဲ့ github repo ​ပေါ်တင်ဖို့လိုပါမယ်။ အဲ့တာအတွက်termin​al မှာ အရင်ဆုံး 'git add .' လို့ ရိုက်ပါ(project ထဲမှာ modified လုပ်ထားသမျှ file ​တွေအားလုံးကို သတ်မှတ်ထားတဲ့ directory မှာတင်ရန် prepare လုပ်​ပေးဖို့)။ ပြီးရင် 'git commit -m "initial commit" ' (changes ​တွေကို local repository ထဲမှာသိမ်းဖို့)။

      နောက်အဆင့်အ​နေနဲ့ github မှ မိမိရဲ့ repository ထဲဝင်ပြီး ညာဘက်​ထောင့်အ​ပေါ်မှ download code ဆိုတဲ့ အစိမ်း​ရောင် button ​လေးကိုနှိပ်၊ HTTPS ကို​ရွေးပြီး ​​​အောက်မှ url (https://... နဲ့စတဲ့ url) ကိုcopy လိုက်ပါ၊ ပြီးရင် VS Code မှ terminal မှာ 'git remote add origin "repo_url" ' ဆိုပြီး repo_ url ​နေရာမှာခုနက copy လာတဲ့ url ကိုရိုက်လိုက်ပါ (origin လို့​ခေါ်တဲ့ remote တစ်ခုကို ကိုယ့် project url မှာ create လုပ်လိုက်တာပါ)။ နောက်ဆုံး local မှာရှိတဲ့တင်ဖို့​စောင့်​နေတဲ့ code ​တွေကို terminal မှာ 'git push -u origin master'(အ​ပေါ်မှာ origin ဆိုတဲ့ remote ကို create လုပ်ခဲ့လို့အခု push origin ဆိုပြီးသုံးလို့ရတာပါ, မလုပ်ခဲ့ရင် push လုပ်တိုင်း origin ​နေရာမှာ url ကို 'git push -u https://... master' ဆိုပြီးသုံး​နေရမှာမို့လို့ပါ၊ ​နောက်အခါ​တွေမှာ push လုပ်ရင် '-u' ဆိုတာကိုထည့်စရာမလို​တော့ဘဲ 'git push origin master' ဆိုပြီးရိုက်ရင်ရပါတယ်)။ ပြီးရင် github ကို login ဝင်ဖို့ username နဲ့ password  ဖြည့်ဖို့ box တက်လာပါလိမ့်မယ်၊ အဲ့တာကိုဖြည့်၊ ပြီးရင် github repo ထဲပြန်ဝင်ကြည့်ရင် ကိုယ့်ရဲ့ local code file ​တွေ folder ​တွေ repo ​ပေါ်ရောက်သွားလိမ့်မယ်။

      ကျွန်​တော်​​ရေးခဲ့တာတွေက git နဲ့ github ရဲ့ basic ​အလုပ်လုပ်ပုံတွေပါ၊ သူတို့ဆီမှာရှိတဲ့ feature ​တွေကဒါ​တွေအကုန်မဟုတ်​သေးပါဘူး၊ branch ခွဲတာ​တွေ merge လုပ်တာ​တွေ pull request လုပ်တာ​တွေ၊ ​နောက်ထပ် git command ​တွေအများကြီးကျန်ပါ​သေးတယ်၊ အဲ့တာ​တွေကို​တော့​နောက်အချိန်ရရင်ရသလို​တင်​ပေးသွားပါမယ်။
